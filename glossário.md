# Glossário ou vocabulário da disciplina

Neste arquivo, são armazenadas palavras ou expressões estudadas na disciplina, com uma breve descrição delas.

---

# Palavras ou Expressões

## CaaS
**Credit as a Service** (Crédito como Serviço), um modelo onde empresas não financeiras oferecem crédito próprio (empréstimos/financiamentos) via APIs, sem precisar de licença bancária.

---

## CRUD
É um acrônimo para as quatro operações básicas de armazenamento persistente em sistemas de banco de dados:

- Create (Criar)
- Read (Ler/Consultar)
- Update (Atualizar)
- Delete (Excluir)

---

## DaaS
O usuário acessa uma área de trabalho completa (Windows ou Linux) através de um navegador ou um app leve em qualquer dispositivo (tablet, PC antigo ou celular).

### Exemplo
- Windows 365

---

## DBaaS
É o modelo onde você “aluga” apenas a funcionalidade do banco de dados, sem precisar gerenciar o servidor, sistema operacional ou instalação do software.

### Exemplo
- Microsoft Azure

---

## FaaS
Popularmente conhecido como **Serverless** (Computação sem Servidor). É ativado quando eventos específicos acontecem, ou seja, o servidor não fica ativo 24 horas por dia.

### Exemplo
- Google Cloud

---

## IaaS
Modelo de computação em nuvem que deixa toda a infraestrutura manual e editável pelo comprador.

### Exemplo
- Microsoft Azure

---

## IoT (Internet das Coisas)
Refere-se a uma rede de objetos físicos equipados com sensores, softwares e tecnologias capazes de conectar e trocar dados pela internet.

### Exemplo
- Alexa

---

## PaaS
Modelo de computação em nuvem que oferece ambiente pronto para desenvolvedores.

### Exemplo
- Google App Engine

---

## SaaS
Software pronto para uso via navegador ou aplicativo. O usuário apenas utiliza o sistema, sem acesso ao código ou servidor.

### Exemplo
- Netflix

---

## Sistemas Pervasivos
Sistema fixo no ambiente que coleta informações do usuário para oferecer melhor experiência possível.

### Exemplos
- Ar-condicionado inteligente
- Alexa

---

## Sistemas Ubíquos
Sistema presente no cotidiano, podendo estar em relógios, roupas ou móveis. Diferente do sistema pervasivo, ele busca ser menos perceptível no dia a dia.

---

## SPT
**Sistemas de Processamento de Transações**. Responsáveis por registrar e processar operações do cotidiano de uma organização.

### Exemplo
- Sistema de vendas do Walmart

---

## SIG
**Sistemas de Informação Gerencial**. Organizam e transformam dados operacionais em informações úteis para gestores.

### Exemplo
- Relatórios gerenciais usados pela Ambev

---

## SAD
**Sistemas de Apoio à Decisão**. Auxiliam na tomada de decisões complexas através da análise de dados e simulação de cenários.

### Exemplo
- Sistemas de análise de dados da Netflix

---

## SIE
**Sistemas de Informação Executiva**. Voltados para executivos, apresentando informações estratégicas por gráficos e dashboards.

### Exemplo
- Dashboards usados por executivos da Google

---

## SE
**Sistemas Especialistas**. Utilizam inteligência artificial para simular conhecimento humano especializado.

### Exemplo
- IBM Watson Health

---

## ERP
**Enterprise Resource Planning** (Planejamento de Recursos Empresariais). Integra todos os setores de uma empresa em um único sistema.

### Exemplo
- ERP utilizado pela Nestlé

---

## CRM
**Customer Relationship Management** (Gestão de Relacionamento com o Cliente). Utilizado para organizar contatos, acompanhar vendas e melhorar o atendimento.

### Exemplo
- CRM utilizado pela Vivo

---

## SISTEMA
Conjunto de elementos que interagem entre si com o objetivo de solucionar problemas e fazer um processo funcionar.

---

## SECaaS
Terceirização da segurança digital para um provedor em nuvem.

### Exemplo
- Cloudflare

# Atividade Aula 14 - VM e VENV

## VM
Uma VM (**Virtual Machine / Máquina Virtual**) é um computador simulado que roda dentro de outro computador físico. Possui sistema operacional, memória, disco e rede próprios.

- Máquina física = hospedeiro
- Máquina virtual = hóspede

---

## VENV
Ferramenta do Python usada para criar ambientes virtuais isolados para projetos.

Permite:
- instalar bibliotecas separadamente
- evitar conflitos entre projetos
- usar versões diferentes de dependências

---

# Atividade Aula 13 - CRM

## CRM
CRM (**Customer Relationship Management**) é uma ferramenta usada para gerenciar o relacionamento com clientes.

### Funções
- organizar contatos
- acompanhar vendas
- registrar atendimentos
- controlar negociações
- automatizar processos

# Atividade Aula 8 - Metodologia Vs Técnica

## Metodologia
Explicação detalhada da abordagem de pesquisa e estudos. Funciona como um plano que organiza o caminho a ser seguido.

---

## Técnica
Conjunto de procedimentos, métodos ou recursos usados para realizar uma atividade específica com eficiência.

Diferença:
- metodologia = planejamento do caminho
- técnica = ferramenta utilizada no caminho

---

## Trello X GitHub
Sim, é possível integrar Trello com GitHub utilizando o Power-Up oficial do GitHub no Trello.

### Permite
- vincular issues
- conectar pull requests
- acompanhar atualizações sem sair do Trello

# Atividade Aula 7 - Modelo TCP/IP

## Modelo TCP/IP
Conjunto de protocolos que define como dispositivos se comunicam na internet.

### Camadas
1. Camada de Aplicação
2. Camada de Transporte
3. Camada de Internet
4. Camada de Acesso à Rede

### Problemas
- falta de criptografia nativa
- vulnerabilidades causadas por softwares desatualizados

### Soluções
- atualização constante dos sistemas
- uso de criptografia na transmissão de dados
